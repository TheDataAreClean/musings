A good tool disappears.

This is the oldest observation about tools and it remains the truest. You don't think about a hammer when you're driving a nail well. You think about the nail, the angle, the depth. The hammer is a condition of the work, not its subject.

## Software Tools Are Different

Software resists disappearing. It updates. It adds features. It asks you to make decisions at the wrong moment — when you're trying to think about something else.

The modal dialog is the emblem of this problem. You are writing. The software interrupts: *do you want to autosave?* You were not thinking about autosaving. You were thinking about the next sentence. The question is not wrong — it is asked at the wrong time. It makes the tool visible.

## The Grain of the Tool

Every tool has a grain. With it, work is easy. Against it, work fights you.

Word processors have a grain. Their grain is: sequential text, styled locally, saved as a single file. When you work with this grain — writing a letter, editing a report — the tool disappears. When you work against it — trying to typeset a book, coordinate with collaborators — the tool becomes a constant presence, a series of workarounds.

The question to ask of any tool: *what is its grain?* And then: *is my work the same shape?*

If yes, the tool disappears. If no, you are fighting it forever.

## On Choosing

I have spent time choosing tools. Too much time, probably. There is a version of this that is pure procrastination — *I will write when I find the right writing environment* — and I have lived in that version.

But there is also a genuine question here. The tool shapes the work. Not in deterministic ways, but in tendencies. A tool that makes revision easy produces different writing than one that makes revision hard. A tool that shows you word count changes how you relate to length.

This is worth thinking about, once, carefully, and then stopping.

---

Choose tools that disappear. Use them until they do. Then write the thing.
